# Demo Script

This is the planned flow for the final demonstration.

## Setup

Use seeded accounts:

- Admin
- Lecturer
- Student

## Demo Flow

1. Show the ERD and explain the main relationships.
2. Log in as admin.
3. Show departments, students, lecturers, courses, sessions, semesters, and course offerings.
4. Show that a course offering belongs to a semester and course.
5. Log in as student.
6. Register courses for the current semester.
7. Show that duplicate registration is blocked.
8. Log in as lecturer.
9. Upload result for registered students.
10. Show grade and grade point are generated.
11. Return to student account.
12. View semester results and transcript.
13. Show SQL view/function/trigger snippets in the presentation.

## Database Concepts To Call Out

- Foreign keys prevent invalid results.
- Unique constraints prevent duplicate registration.
- Views generate transcript output.
- Functions compute GPA and CGPA.
- Triggers classify grades and write audit logs.
- Transactions protect course registration and result upload.
